65.選擇器:
    1.標籤選擇器
    2.id選擇器:根據指定的id名稱找到對應的標籤設置屬性,id名稱具唯一性,並以#id名稱(只能由字母/數字/下劃線組成,不能以數字開頭),一般id名稱是留給js使用,一個HTML標籤只能綁定一個id名稱。

    6.交集選擇器:給所有選擇器選中的標籤中,相交的部份標籤設置屬性,之間無空格連接
    7.並集選擇器:給所有選擇器選中的標籤設置屬性,標籤之間以,隔開,主要用來簡化CSS設置重複的代碼
    8.兄弟選擇器:只能選中同級關係
      (1)相鄰兄弟:給指定選擇器後面緊跟的那個選擇器選中的標籤設置屬性,選擇器之間以+連接,限相鄰兄弟標籤(選擇器)才會被選中
      (2)通用兄弟:給指定選擇器後面的所有選擇器選中的所有標籤設置屬性,選擇器之間以~連接,不限相鄰與否
    9.同級別選擇器(CSS3新增)-序選擇器
      (1):first-child:選中同級別中的第一個標籤,不區分類型
      (2):last-child:選中同級別中的最後一個標籤,不區分類型
      (3):nth-child(n):選中同級別中的第n個標籤,不區分類型,n->even 偶數 n->odd 奇數 , 2n -> 偶數, 2n+1 -> 奇數
      (4):nth-last-child(n):選中同級別中的倒數第n個標籤,不區分類型
      (5):only-child:選中父元素中只有一個子元素的元素
      (6):first-of-type:選中同級別中同類型的第一個標籤,區分類型
      (7):last-of-type: 選中同級別中同類型的第一個標籤,區分類型
      (8):nth-of-type(n):選中同級別中同類型的第n個標籤,區分類型
      (9):nth-last-of-type(n):選中同級別中同類型的倒數第n個標籤,區分類型
      (10):only-of-type:選中父元素中只有一個同類型的某個標籤,區分類型
      n->even 偶數  2n
      n->odd 奇數 2n+1 
      3n -> 第3倍數個  5n -> 第5倍數個
      n+3 -> 前數3個 -n+3 -> 倒數3個
    10.屬性選擇器:根據指定的屬性名稱找到對應的標籤,然後設置屬性
      (1)[attr]:根據指定的屬性名稱找到對應的標籤,然後設置屬性
      (2)[attr=value]:找到指定屬性且屬性值=value的標籤,然後設置屬性,最常見的應用場景,用於區分input屬性
      (3)[attr^=value]:找到屬性值以value開頭
      (4)[attr$=value]:找到屬性值以value結尾
      (5)[attr*=value]:找到屬性值包含value
    11.通配符選擇器:*-->給當前界面上所有的標籤設置屬性,會降低性能,所以在企業開發中不會使用通配符選擇器
83.CSS三大特性:
  1.繼承性:給父元素設置的某些屬性,子元素也可以使用,並不是所有屬性都可繼承,只有以color/font-/text-/line開頭的屬性才可繼承,在CSS中不僅子元素可繼承,後代元素均可繼承,CSS繼承中的注意點:
     (1)a標籤中的文字顏色和下劃線不能繼承
     (2)h標籤的文字大小也不能繼承
    應用場景:一般用於設置網頁上的一些共同屬性,如網頁的文字顏色、字體、文字大小等內容
    body{
        font:16px
    }
  2.層疊性:CSS處理衝突的一種能力,只有在多個選擇器選中"同一個標籤",且設置了"相同的屬性",才會發生層疊性。
  3.優先級:當多個選擇器同一個標籤,且設置相同的屬性時,如何層疊就由優先級來確定,優先級判斷的三種方式:
    (1)間接選中(即是繼承),如果是間接選中(繼承而來),則採標籤就近原則(誰離目標元素比較近就採用誰的)
    (2)相同選擇器,如果都是直接選中,且都是同類型的選擇器,依代碼就近原則,誰寫在後面就採用誰的
    (3)不同選擇器,如果都是直接選中,且不是相同類型的選擇器,就會按照選擇器的優先級來層疊
    優先級順序:  id > 類(偽類) > 標籤(元素、偽元素) > 通配符 > 繼承 > 瀏覽器默認
86.優先級提昇!important:只能用於直接選中,不能用於間接選中,只能提昇被指定的屬性的優先級,其他的屬性的優先級不會被提昇。
87.優先級的權重:當多個選擇器混合在一起使用時,可通過計算權重來判斷誰的優先級最高
    1.權重計算規則:
      (1)先計算選擇器中有多少個id,id多的選擇器優先級最高
      (2)如果id的個數相同,再看類名的個數,類名多的選擇器優先級較高
      (3)如果類名的個數相同,則看標籤名的個數,標籤名個數多的優先級較高
      (4)如果id、類名及標籤名個均相同時,採就近原則,誰寫在後面就採用誰的
    2.id     1 0 0 0
      class  0 1 0 0
      標籤名  0 0 1 0
      通配符  0 0 0 1
      繼承來  0 0 0 0
      混合使用時直接計算相加但不可跨級進位
89.div和span標籤
   div作用:用來配合CSS完成網頁的基本布局
   span作用:用於配合CSS修改網頁中的一些局部信息
   div和span的區別:
   (1)div會獨佔一行,而span不會獨佔一行
   (2)div是一個容器級標籤,而span是一個文本級標籤
   **容器級標籤可以嵌套其他所有標籤,而文本級標籤只能嵌套文字/超鏈接/圖片,不能嵌套容器級標籤
   **容器級標籤:div h ul li ol dl..
   **文本級標籤:span p em b u i s strong del ins...
90.CSS元素顯示模式
    在HTML中將所有標籤分為兩類:容器級、文本級標籤
    在CSS中將所有標籤分為兩類:塊級元素、行內元素
    塊級元素:會獨佔一行,所有容器級元素和p標籤
    行內元素:不會獨佔一行,除了p以外所有文本級元素均是
    區別:
    **塊級元素:
        1.獨佔一行,而行內元素不會獨佔一行
        2.如果未設置寛度,則默認和父元素一樣寛
        3.可設置寛高
    **行內元素:
        1.不會獨佔一行
        2.如果未設置寛高,則默認和文本內容一樣寛
        3.不能設寛高
    **行內塊級元素:不會獨佔一行,且可設置寛高
91.CSS元素顯示模式轉換
    利用display設置inline(行內級)/block(塊級)/inline-block(行內塊級)
    display:inline/block/inline-block;

95.背景平舖:background-repeat
    默認在水平/垂直方向平舖,
    no-repeat 水平/垂直均不平舖
    repeat-x  只在水平方向平舖
    repeat-y  只在垂直方向平舖
    應用場景:利用背景圖片平舖來降低圖片的大小,提昇網頁的訪問速度
96.背景定位:background-position: 水平方向位置 垂直方向位置;默認在左上角(0 0)顯示
    1.同一個標籤可以同時設置背景圖片和背景顏色,如果同時設置時,圖片會覆蓋顏色
    2.取值:具體方位名詞/具體像素值(記得要加上單位px,也可以是負值)
    3.應用:使用大圖再利用background-position:center;設定使其居中顯示,無論放大或縮小窗口均可居中顯示重要內容。
98.背景縮寫
    1.格式:background: 背景顏色 背景圖片 平舖方式 關聯方式 定位方式
    2.注意點:
      (1)任何一個屬性均可省略
      (2)背景關聯方式:用於設置背景圖是否隨滾動條滾動,默認scroll(會跟隨滾動)
         background-attachment:scroll(默認值,背景會隨滾動條滾動)/fixed(不會隨滾動條滾動)
99.背景圖片和插入圖片的區別:
    1.背景圖片只是一個裝飾,不會佔用位置,而插入圖片會佔用位置。
    2.背景圖片有定位屬性,可控制圖片擺放位置,而插入圖片無法有效控制圖片位置
    3.插入圖片的語義比背景圖片的語義強,如果圖片要被搜索引擎收錄,推薦使用插入圖片。

110.盒子的寛/高
    1.內容的寛/高:通過標籤的width/height屬性設置的寛度和高度
    2.元素的寛/高:包括左邊框+左內邊距+width+右內邊距+右邊框
    3.元素空間的寛/高:包括左外邊距+左邊框+左內邊距+width+右內邊距+右邊框+右外邊距
112.
    1.padding(內邊距)和border(邊框)設置大於0的值時均會使元素的寛/高加大。
    2.設置box-sizing:content-box;默認值
      box-sizing:border-box;CSS3新增box-sizing屬性,用來保證元素的寛/高度不變,不會因置padding/border值後撐大盒子,即元素的寛/高=width/height
114.盒子水平居中
    1.如果兩個盒子是嵌套關係(父子),只要設置子元素盒子的頂部外邊距(margin-top)屬性,父元素盒子也會一起往下移(問題點)。
    2.如果要解決父元素盒子不會受子元素盒子設置margin-top值的影響而往下移問題,可在父元素盒子添加border邊框屬性即可。
    3.在企業開發,一般情況下如果要控制嵌套(父子)關係盒子之間的距離,首先考慮使用padding,其次再考慮使用margin,因為margin本質上是用來控制兄弟關係之間的間隙的。
    4.在嵌套父子關系的盒子中,可利用margin:0 auto;來使子元素盒子在父元素盒子中水平居中,margin:0 auto;只對水平方向有效,對垂直方向無效。
115.盒子水平居中(margin:0 auto;)和內容居中(text-align:center;)的區別
    1.text-align:center;作用:用來設置盒子中存儲的巾容(文字/圖片)水平居中
    2.margin:0 auto;作用:讓盒子本身水平居中
116.清空默認邊距
    1.為什麼要清空默認邊距?在開發中為了準確的控制和計算盒子的寛高,因此在編寫代碼前會先清空瀏覽器及HTML標籤自帶的默認邊距,可在CSS中添加下面代碼,但實際開發中不建議如此用,因為較佔資源,可引入初始化CSS
    *{
        margin:0;
        padding:0;
    }      
117.行高和字號
    1.行高:line-height  行高與盒子高是不同的概念
      (1)盒子高度(height)未設置時,默認由文字內容撐開  
      (2)文字在行高中默認是垂直居中的
      (3)將盒子高度和行高設置相同值時,就可保證一行文字在盒子中垂直居中
      (4)盒子中有多行文字,不能使用設置行高等於盒子高度來實現文字在盒子中垂直居中,只能透過設置padding來讓文字居中
      (5)如果盒子中存儲的是文字,一般會以盒子左邊的內邊距為基準,不會以右邊的內邊距為基準,因為右邊距通常會有誤差(因為右邊如果放不下一個文字,文字會換行顯示,所以文字和內邊距之間的距離就會有誤差)
      (6)頂部的內邊距並不是邊框到文字頂部的距離,而是邊框到行高頂部的距離
120.網頁布局方式:意即瀏覽器如何對網頁中的元素進行排版,常見三種排版方式
	1.標準流(文檔流/普通流):瀏覽器默認排版方式,在CSS中將元素分為三種:塊級/行內/行內塊元素,有兩種排版方式:
	  (1)垂直排版:塊級元素,由上到下垂直排版
	  (2)水平排版:行內元素,由左至右水平排版
	2.浮動流:是一種半脫離標準流的排版方式,只有水平排版方式,只能設置某個元素靠左/靠右對齊,没有居中對齊方式
	  float:none/left/right; <--没有center
	  注意點:
	  (1)不可使用margin: 0 auto;
	  (2)不區分塊級/行內/行內塊級元素,且均能水平排版及設置寛高
	  (3)意即浮動流中的元素和標準流中的行內塊級元素類似
	3.定位流
122.浮動元素的脫標(脫離標準流):當某一個元素設置浮動後,則這個元素看上去就像從標準流中被刪除一樣(脫標),不繼續佔有原來位置,則下面的元素會往上提佔有原來位置,而形成上下層關係。
123.浮動元素的排序規則:
	1.相同方向上的浮動元素,先浮動的元素會顯示在前面,後浮動的元素會顯示在後面
	2.不同方向上的浮動元素,左浮動會找左浮動,右浮動會找右浮動。
	3.浮動元素浮動後的位置,由浮動元素浮動前在標準流中的位置來確定
124.浮動元素貼靠現象
    1.如果父元素盒子的寛度足够顯示所有浮動元素時,則浮動元素會併排顯示在父元素盒子中
    2.當父元素盒子的寛度不足以顯示所有浮動元素時,則會從最後一個元素開始往前貼靠。
    3.當貼靠了前面所有浮動元素後都不能顯示時,最終會貼靠到父元素的左邊或右邊。
125.浮動元素字圍現象(浮動本質是做文繞圖、圖文混排效果)
    浮動元素不會將蓋住文字,而是環繞在浮動元素後面開始排列產生文繞圖效果。
129.浮動元素高度問題
    1.在實際開發中,通常垂直方向使用標準流,水平方向使用浮動流
    2.如何布局頁面?通常採用從上到下從外而內的思考模式。
    3.水平方向可先劃分一左一右再對左邊或右邊進行進一步布局
	4.高度問題:
	  (1)在標準流中內容的高度會撐起父元素的高度
	  (2)在浮動流中,浮動元素是不可以撐起父元素的高度而產生高度塌陷問題
130.清除浮動方式
    1.給前面一個父元素設置固定高度,但在實際開發中,能不寫高度就不寫高度,所以此種方式很少用
    2.給後面的盒子添加clear屬性,取值有
      none:默認值,按照浮動元素的排序規則來排序
      left:不要找前面的左浮動元素
      right:不要找前面的右浮動元素
      both:不要找前面的左浮動元素和右浮動元素
      注意點:當我們給某個元素添加clear屬性之後,則這個元素的margin屬性會失效
    3.隔牆法:
      (1)外牆法:在兩個盒子中間添加一個額外的塊級元素,並設置clear:both屬性,但如此做法會造成第一個盒子不能使用margin-bottom屬性,如要加margin做法通常在額外添加的塊級元素中添加高度隔開。
      (2)內牆法:在第一個盒子中所有子元素最後添加一個額外的塊級元素,並設置clear:both屬性,不會造成第一個盒子不能使用margin-bottom屬性現象
      兩者區別:
       1.外牆法不能撐起第一個盒子的高度,而內牆法可以撐起第一個盒子的高度
       2.外牆法會造成第一個盒子不能使用margin-bottom屬性,而內牆法不會
      兩者共同點:均要添加額外的標籤,實際開發中不常用隔牆法來清除浮動。
    4.利用在第一個盒子添加偽元素選擇器-->推薦使用
      ::after{
        content:"";
        display:block;
        height:0;
        visibility:hidden;
        clear:both;
      }
      IE6時記得再添加下面一行
      .box1{
      *zoom:1;
      }
135.偽元素選擇器:CSS3新增,作用:給指定標籤的內容前面/或後面添加一個子元素
    1.::before {
        content:"";   //要添加的內容,無則空白,此一屬性必需寫
        visibility:hidden;  //可將內容隠藏
        display:block;     //改變顯示模式為塊級元素,可設置寛高
    }
    2.::after {
        contentL"";    //要添加的內容,無則空白,此一屬性必需寫
    }
137.overflow:hidden 作用
    1.可將超出標籤範圍的內容裁剪掉
    2.清除浮動
    3.可通過設置overflow:hidden讓嵌套關係中子元素盒子設置margin-top之後,父元素盒子不會被頂下來
