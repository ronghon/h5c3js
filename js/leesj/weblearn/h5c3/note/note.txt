65.選擇器:

    2.id選擇器:根據指定的id名稱找到對應的標籤設置屬性,id名稱具唯一性,並以#id名稱(只能由字母/數字/下劃線組成,不能以數字開頭),一般id名稱是留給js使用,一個HTML標籤只能綁定一個id名稱。

    7.並集選擇器:給所有選擇器選中的標籤設置屬性,標籤之間以,隔開,主要用來簡化CSS設置重複的代碼

      (1)相鄰兄弟:給指定選擇器後面緊跟的那個選擇器選中的標籤設置屬性,選擇器之間以+連接,限相鄰兄弟標籤(選擇器)才會被選中
      (2)通用兄弟:給指定選擇器後面的所有選擇器選中的所有標籤設置屬性,選擇器之間以~連接,不限相鄰與否

      (1):first-child:選中同級別中的第一個標籤,不區分類型
      (2):last-child:選中同級別中的最後一個標籤,不區分類型
      (3):nth-child(n):選中同級別中的第n個標籤,不區分類型,n->even 偶數 n->odd 奇數 , 2n -> 偶數, 2n+1 -> 奇數
      (4):nth-last-child(n):選中同級別中的倒數第n個標籤,不區分類型
      (5):only-child:選中父元素中只有一個子元素的元素
      (6):first-of-type:選中同級別中同類型的第一個標籤,區分類型
      (7):last-of-type: 選中同級別中同類型的第一個標籤,區分類型
      (8):nth-of-type(n):選中同級別中同類型的第n個標籤,區分類型
      (9):nth-last-of-type(n):選中同級別中同類型的倒數第n個標籤,區分類型
      (10):only-of-type:選中父元素中只有一個同類型的某個標籤,區分類型
      n->even 偶數  2n
      n->odd 奇數 2n+1 
      3n -> 第3倍數個  5n -> 第5倍數個
      n+3 -> 前數3個 -n+3 -> 倒數3個
    10.屬性選擇器:根據指定的屬性名稱找到對應的標籤,然後設置屬性
      (1)[attr]:根據指定的屬性名稱找到對應的標籤,然後設置屬性
      (2)[attr=value]:找到指定屬性且屬性值=value的標籤,然後設置屬性,最常見的應用場景,用於區分input屬性
      (3)[attr^=value]:找到屬性值以value開頭
      (4)[attr$=value]:找到屬性值以value結尾
      (5)[attr*=value]:找到屬性值包含value

83.CSS三大特性:
  1.繼承性:給父元素設置的某些屬性,子元素也可以使用,並不是所有屬性都可繼承,只有以color/font-/text-/line開頭的屬性才可繼承,在CSS中不僅子元素可繼承,後代元素均可繼承,CSS繼承中的注意點:
     (1)a標籤中的文字顏色和下劃線不能繼承
     (2)h標籤的文字大小也不能繼承
    應用場景:一般用於設置網頁上的一些共同屬性,如網頁的文字顏色、字體、文字大小等內容
    body{
        font:16px
    }
  2.層疊性:CSS處理衝突的一種能力,只有在多個選擇器選中"同一個標籤",且設置了"相同的屬性",才會發生層疊性。
  3.優先級:當多個選擇器同一個標籤,且設置相同的屬性時,如何層疊就由優先級來確定,優先級判斷的三種方式:
    (1)間接選中(即是繼承),如果是間接選中(繼承而來),則採標籤就近原則(誰離目標元素比較近就採用誰的)
    (2)相同選擇器,如果都是直接選中,且都是同類型的選擇器,依代碼就近原則,誰寫在後面就採用誰的
    (3)不同選擇器,如果都是直接選中,且不是相同類型的選擇器,就會按照選擇器的優先級來層疊
    優先級順序:  id > 類(偽類) > 標籤(元素、偽元素) > 通配符 > 繼承 > 瀏覽器默認
86.優先級提昇!important:只能用於直接選中,不能用於間接選中,只能提昇被指定的屬性的優先級,其他的屬性的優先級不會被提昇。
87.優先級的權重:當多個選擇器混合在一起使用時,可通過計算權重來判斷誰的優先級最高
    1.權重計算規則:
      (1)先計算選擇器中有多少個id,id多的選擇器優先級最高
      (2)如果id的個數相同,再看類名的個數,類名多的選擇器優先級較高
      (3)如果類名的個數相同,則看標籤名的個數,標籤名個數多的優先級較高
      (4)如果id、類名及標籤名個均相同時,採就近原則,誰寫在後面就採用誰的
    2.id     1 0 0 0
      class  0 1 0 0
      標籤名  0 0 1 0
      通配符  0 0 0 1
      繼承來  0 0 0 0
      混合使用時直接計算相加但不可跨級進位
