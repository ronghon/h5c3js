01.課程簡介
    C/S架構軟件:C客戶端、S服務器
     - 客戶端:用戶端通過軟件執行
     - 服務器:用來處理軟件的業務邏輯
     - 例子:QQ、office、王者榮耀...等。
     - 特點:
        1.C/S架構軟件必須安裝後才能使用。
        2.C/S架構軟件更新時,服務器和客戶端都需更新。
        3.C/S架構的軟件無法跨平台使用。
        4.C/S架構的軟件客戶端和服務器間的通信採用自有的協議,安全性較好。
    B/S架構軟件:B瀏覽器,S服務器
     - 本質上也是C/S,B/S使用瀏覽器作為軟件的客戶端
     - B/S實際上就是一個一個的網站,可通過訪問一個網站來使用軟件
     - 例子:京東、淘寶...等
     - 特點:
        1.不需要安裝可以直接使用
        2.軟件更件客戶端無需更新
        3.軟件可跨平台使用
        4.B/S架構的軟件,客戶端和服務器間的通信採用公共的http協議,安全性較差。

16.語義化標籤
    在網頁中HTML專門負責網頁的結構,在使用HTML標籤時,應關注標籤的語義,而不是它的樣式。
    1.標題標籤:從h1~h6重要性遞減,其中h1最重要,h6最不重要,h1在網頁中的重要性僅次於title標籤,從SEO角度而言,一般
      情況下,一個頁面只會有一個h1標籤,是一個塊級元素
    2.塊級元素:在頁面中會獨佔一行的元素。(block element)
    3.段落標籤:<p>表示頁面中的一個段落,也是塊級元素
    4.em標籤:表示加重語音語調,行內元素,在頁面中不會獨佔一行的元素稱為行內元素(inline element)
    5.strong:表示強調重要內容,字體會加粗
    6.blockquote:長引用,塊元素 q:短引用,行內元素
    7.br 換行, hr 分割線

17.塊級元素及行內元素
    1.塊元素(block element):在網頁中一般通過塊元素進行布局用
      (1)一般情況下會在塊元素中嵌套行內元素,而不會在行內元素中嵌套塊元素
      (2)塊元素中基本上可以嵌套任意元素,但p元素中不能嵌套任何塊元素
    2.行內元素(inline element):主要用來包裏文字
    3.瀏覽器解析網頁時,會自動對網頁中不符合規範的內容進行修正,ex:
      (1)標籤寫在根元素的外部
      (2)p元素嵌套塊元素
      (3)根元素出現除了head和body以外的子元素

27.CSS簡介:層疊樣式表,用來設置網頁中元素的樣式,網頁實際上是一個多層的結構,通過 CSS 可以分別為網頁的每一層設置樣式,而最後呈現在客戶端瀏覽器的是最上一層,CSS 編寫位置區分為三種: 
    1.行內(內聯)樣式:<p style="color:red; font-size:16px;">標題</p>
      接在標籤中使用 style 寫在行間,其中屬性以鍵值對方式書寫,並以;結尾,樣式一次僅能對單一標籤生效,後續樣式修改時需一個一個改,對後期維護不易,所以不建議使用此方式開發。 
    2.內部樣式:
      <style>
          p {
           color: red; 
           font-size: 16px;  
          }
      </style>
      在 head 標籤中加入 style 標籤並以選擇器{}形式選中元素並為其設置各種樣式,可以同時為多個標籤設置樣式且後續修改時僅需修改一處即可為選中的標籤全部同時修改,方便後續修改,但內部樣式表只能對當前頁面起作用,不能跨頁面進行重複使用。
    3.外部樣式表:另外創建一個.css檔案,再利用link標籤引入,只要使用相同樣式都可以進行引用,方便後續維護與不同頁面間重用,推薦使用,可使用瀏覽器的緩存機制,從而加快網頁的加載速度,提高用戶體驗。
        <link rel="stylesheet href="index.css">

30.常用選擇器:
    1.元素選擇器
      作用:根據標籤名來選中指定的元素
      語法:標籤名{}
      例子:p{}  h1{}  div{}
    2.id選擇器:#開頭id不可重複
      作用:根據元素的id屬性值選中一個元素
      語法:#id屬性值{}
      例子:#box{} #red{}  
    3.類選擇器:.開頭
      作用:根據元素的class屬性值選中一組元素,使用.來表示class,class值可重複,可以同時設置多個class值,之間以空格隔開。
      語法:.class屬性值{}
    4.通配符選擇器
      作用:選中頁面中的所有元素      
      語法: *{}

31.複合選擇器
    1.交集選擇器:選擇器之間連著寫無空格,and關係
      作用：選中同时复合多個条件的元素
      语法：選擇器1選擇器2選擇器3選擇器n{}
      注意點：
          交集選擇器中如果有元素選擇器，必须使用元素選擇器開头
    2.選擇器分组（并集選擇器）:or關係
      作用：同时選擇多個選擇器對應的元素,選擇器之間以逗號隔開
      语法：選擇器1,選擇器2,選擇器3,選擇器n{}

32.父子兄弟關係選擇器
    1.父元素 - 直接包含子元素的元素叫做父元素
    2.子元素 - 直接被父元素包含的元素是子元素
    3.祖先元素
      - 直接或間接包含後代元素的元素叫做祖先元素
      - 一個元素的父元素也是它的祖先元素
    4.後代元素
      - 直接或間接被祖先元素包含的元素叫做後代元素
      - 子元素也是後代元素
    5.兄弟元素 - 擁有相同父元素的元素是兄弟元素
    ----------------------------------------
    1.子元素選擇器:使用 > 連接
      作用：選中指定父元素的指定子元素
      语法：父元素 > 子元素
    2.後代元素選擇器:使用空格連接
      作用：選中指定元素内的指定後代元素
      语法：祖先 後代
    3.選擇下一個兄弟
      语法：前一個 + 下一個
    4.選擇下邊所有的兄弟
      语法：兄 ~ 弟

34.偽類和偽元素選擇器
   偽類（不存在的類，特殊的類）,用來描述一個元素的特殊状态,比如：第一個子元素、被点击的元素、鼠标移入的元素...,一般情况下都是使用:開頭
    1. :first-child 第一個子元素
    2. :last-child 最後一個子元素
    3. :nth-child(n) 選中第n個子元素
    4. :only-child
      特殊值：
        n 第n個 n的範圍從0到正無窮
        2n 或 even 表示選中偶数位的元素
        2n+1 或 odd 表示選中奇数位的元素
    - 以上这些偽類都是根據所有的子元素进行排序
    5. :first-of-type
    6. :last-of-type
    7. :nth-of-type()
    8. :only-of-type
    9. :empty
    - 這幾個偽類的功能和上述的類似，不同處是他們是在同類型元素中進行排序
    10. :not() 否定偽類 - 將符合條件的元素從選擇器中去除,除了某個之外的其他元素

38.樣式繼承:我们为一個元素設置的樣式同时也會應用到它的後代元素上
   1.繼承是發生在祖先後代之间的
   2.繼承的設計是为了方便我们的開發，利用繼承我们可以将一些通用的樣式统一設置到共同的祖先元素上，这樣只需設置一次即可让所有的元素都具有该樣式
   3.注意：并不是所有的樣式都會被繼承： 比如 背景相关的，布局相关等的这些樣式都不會被繼承。

39.選擇器權重(優先級)
   1.樣式的衝突 - 当我们通過不同的選擇器，選中相同的元素，并且为相同的樣式設置不同的值时，此时就發生了樣式的衝突。
   2.發生樣式衝突时，應用哪個樣式由選擇器的權重（優先级）决定
     選擇器的權重
     (1)内联樣式      1,0,0,0
     (2)id選擇器      0,1,0,0
     (3)類和偽類選擇器 0,0,1,0
     (4)元素選擇器    0,0,0,1
     (5)通配選擇器    0,0,0,0
     (6)繼承的樣式    没有優先级
   3.比较優先级时，需要将所有的選擇器的優先级进行相加計算，最後優先级越高，则越優先显示
    (1)分组選擇器是单独計算的
    (2)選擇器的累加不會超過其最大的数量级，類選擇器在高也不會超過id選擇器
    (3)如果優先级計算後相同，此时则優先使用靠下的樣式(就近原則)
    (4)可在某個樣式後邊添加 !important，此时该樣式會獲取最高優先级，甚至超内联樣式，注意：在開發開中一定要慎用！

   4.rem:相對單位

44.文檔流(normal flow)
  - 網頁是一個多層的結構,一層壓著一層,通过CSS可分别为每一層設置樣式,用户只能看到最上一層
  - 這些層中,最底下的一層稱為文檔流,文檔流是網頁的基礎,創建的元素默認都在文檔流中進行排列
  - 元素主要有兩個狀態
	(1) 在文檔流中
	(2) 不在文檔流中(脱離文檔流)
  - 元素在文檔流中有什麼特點：
    (1) 塊元素ex:div p h1 li header nav footer
        - 塊元素會在頁面中獨佔一行(自上向下垂直排列)
        - 默認寛度(未設置時)是父元素的全部(會把父元素撑满)
        - 默認高度(未設置)是被内容撑開(子元素)
    (2) 行内元素ex:span i em 
        - 行内元素不會獨占頁面的一行,只占自身的大小
        - 行内元素在頁面中左向右水平排列,如果一行之中不能容纳下所有的行内元素
          则元素會換到第二行繼續自左向右排列(和書寫習慣一致)
        - 行内元素的默認寛度和高度都是被内容撑開

45.盒模型、盒子模型、框模型(box model)
   - CSS將頁面中的所有元素都設置為了一個矩形的盒子
   - 將元素設置為矩形的盒子後，對頁面的布局就變成將不同的盒子擺放到不同的位置
   - 每一個盒子都由以下幾個部分组成：
     内容區(content)
     内邊距(padding)
     邊框(border)
     外邊距(margin)
	1.内容區(content),元素中的所有的子元素和文本内容都在内容區中排列,内容區的大小由width 和 height两個属性来設置
      width 設置内容區的宽度
      height 設置内容區的高度

136.像素
  1.像素：
    - 屏幕是由一個一個發光的小點構成,這一個個的小點就是像(px)
    - 解析度:1920 x 1080 意即屏幕中橫向有1920個縱向有1080個小點所組成
    - 在前端開發中像素分成兩種:CSS像素和物理像素
    - 物理像素,上述所說的小點點就屬於物理像素
    - CSS像素,編寫網頁時，我們所用的像素都是CSS像素
        - 瀏覽器在顯示網頁時,需先將CSS像素轉換為物理像素然後再呈現
        - 一個css像素最终由幾個物理像素顯示,由瀏覽器决定:默認情况下在pc端,一個css像素 = 一個物理像素

  2.視口（viewport）
    - 視口就是屏幕中用来顯示網頁的區域(瀏覽器的窗口)
    - 可以通過查看視口的大小,來觀察CSS像素和物理像素的比值
    - 默認情况下：
        視口寛度 1920px（CSS像素）
                1920px（物理像素）
                - 此時,css像素和物理像素的比是1:1 
    - 放大兩倍的情况：
        視口寛度 960px（CSS像素）
                1920px（物理像素）
                - 此时，css像素和物理像素的比是1:2
    - 我们可以通过改变视口的大小，来改变CSS像素和物理像素的比值
