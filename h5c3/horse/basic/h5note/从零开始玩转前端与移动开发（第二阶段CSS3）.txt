- 選擇器:使用時最多不要超過3級
    -.標籤:元素{},指定單一標籤(元素)
    -.類:class="類名"  .類名{},可多次重用,多類名之間以空格區格,前後順序不會影響
    -.id: id="id名"  #id名{} 具唯一性,同一id名不可重用
    -.通用: *{} * 所有選擇器,通常用於初始化
    -.偽類:為某些選器添加特殊效果,主要針對<a>標籤 注意順序:  LV包包 HA好
        :link{} 尚未點擊過的鏈接
        :visited{}已點擊過的鏈接
        :hover{}鼠標移入到鏈接上時-->常用
        :active{}鼠標點擊選定,鼠標當未鬆開顯示狀態
    -.結構偽類:
        :first-child{}指定選取第一個子元素
        :last-child{}指定選取最後一個子元素
        :nth-child(n)指定選取第n個子元素
        :nth-child(even)偶數,也可使用2n   :nth-child(odd)奇數個,也可使用2n+1,  3n選取3的倍數個
        :nth-last-child(n)從後面往前數第n個子元素
    -.目標偽類:用於選取當前活動的目標元素 誰被選中誰就執行樣式  :target{}
    -.複合選擇器:兩個或多個選擇器時使用,指定目標更精準明確,權重相對提高
      1.交集:兩個選擇器中間無空格(並且,即...又...)
      2.并(聯)集:選擇器中間以,逗號隔開(a,b,c..可集體聲明,a和b和c)
      3.後代:包含父子、孫輩、曾孫輩,兩選擇器之間以空格隔開
      4.子元素:限於父子闗係,兩選擇器之間以 > 隔開
      5.屬性:當某些標籤帶有特殊屬性的選擇器稱之為屬性選擇器 
        (1)E[attr] {} ex: div[title]{}選取屬性名為attr
        (2)E[attr=val]{}選取屬性值為val,不用加分號
        (3)E[attr^=val]{}選取開頭為val元素
        (4)E[attr$=val]{}選取結尾為val元素
        (5)E[attr*=val]{}選取任意位置包含有val元素
      6.偽元素:用來選取元素中的特殊位置
        ::first-letter 只選取第一個字 
        ::first-line 只選取第一行
        ::selection 當鼠標選取文字時,可改變的樣式
        ::after(元素後加上內容,content="內容"-->必要):在盒子內部的後面插入內容 
        ::before (在元素前加上內容,content="內容"-->必要):在盒子內部的前面插入內容

- 塊級元素(div p h1~h6 ul ol li header nav footer...)display:block
  1.可設置寛高,未設置時默認寛度為auto(為父元素的100%),當指定內邊距時不會影響可見框的大小,且會自動修改文本寛度
  2.默認高度被文本內容撐開
  3.獨佔一行,由上到下排列
  4.可容納行內元素及其他塊級元素,但需注意下列嵌套問題
    -標籤嵌套問題:
    1.超鏈接不能嵌套超鏈接
    2.p標籤不能嵌套p標籤,文字類塊級標籤中不能放其他塊級元素
    3.標題h1-h6也不能互相嵌套
- 行內(內聯)元素(span a i em strong..)display:inline;
  1.不會獨佔一行,不能設置寛高,且寛高依文本自身大小設置由左至右排列
  2.可設置水平方向內邊距(padding)及邊框,且設置垂直方向內邊距時不影響其他元素及頁面佈局
  3.可設置水平方向外邊距(margin-left及right),不支持垂直方向外邉距(margin-top及bottom)
  4.行內元素能容納文本和其他行內元素,不可嵌套塊級元素
- 行內塊元素(img input td)display:inline-block;
  1.不會獨佔一行,但可設置寛高,默認寛度為本身內容的寛度
  2.高度、行高、內邊距都可控制

- css三大特性:層疊性、繼承性、優先級
  1.樣式衝突遵循原則:就近原則後面蓋前面,優先級高者優先使用
  2.繼承性:子承父業(限於text- font- line- color...)但並非所有樣式都會繼承,查詢css樣式表
  3.優先級:(權重問題)(選擇器選擇的範圍越小越精確,優先級越高)
    (1)!important > 行內樣式 1000 > id選擇器 100 > 類、屬性、和偽類選擇器 10 > 元素和偽元素選擇器 1 > 通用選擇器* 0 > 繼承的樣式(没有優先級)
    (2)權重可疊加:當選擇器中包含多種選擇器時,需將多種選擇器的權重相加後再作比較
    (3)權重疊加計算時没有進級問題  1,0,0,0(行內)  0,1,0,0(id)  0,0,1,0(類、屬性、和偽類)  0,0,0,1(元素和偽元素) 0,0,0,0 (通用)  繼承(没有優先級)
    (4)如果選擇器優先級相同時,則使用後面的樣式
    (5)并(聯)集的優先級是單獨計算
    (6)!important 優先級最高,慎用!!
    (7)a下的四個偽類選擇器順序:link --> :visited --> :hover --> :active  (LOVE HATE) 
- 盒模型:包括四個部份-->content、padding、border、margin
    1.content
      (1)預設值:box-sizing: content-box;(向外擴展)
		 content實際寬高=width+左右padding值+左右border值
         如果一個盒子没有設置寛、高或繼承父元素的寛、高時,則padding不會影響本身盒子的大小
	  (2)box-sizing: border-box;(向內壓縮)設置變更盒模型的大小計算方式
         content實際寬高=width(巳包含border,padding)
	  前綴:firefox-->-moz- safari-->-webkit-

    2.padding內邊距:指內容到邊框之間的距離,有四個方向可設置,設置順序:上右下左
       一個值:四邊設置相同值,兩個值:上下  左右  三個值 上 左右 下
    3.border邊框
       - border-width | border-style | border-color
       - border-radius邊框圓角
        一個值表示四個角均為相同,設置為50%時會變成圓形
        二個值表示左上角與右上角,右上角與左下角
        三個值表示左上角,右上角與右下角,右下角
    4.margin外邊距:指盒子與瀏覽器邊框之間的距離,有四個方向可設置,設置順序:上右下左
        一個值:四邊設置相同值,兩個值:上下  左右  三個值 上 左右 下
        設置 margin: 0 auto;讓塊級帶有寛度的盒子在瀏覽器水平居中對齊

- 清除默認內外邊距,使用  *{margin:0;padding:0}
- 行內元素只有左右外邊距(margin-left),没有上下外邊距(margin-top、bottom),一般儘量不要給行內元素指定上下的內外邊距
- 相鄰垂直方向的外邊距會發生外邊距重疊問題:兩個塊級盒子均設置垂直方向外邊距時,則以最大的外邊距為準而非取和相加,如有需要,只需設置一個值即可,不用重覆設定
- 相鄰垂直方向之父子元素之間,當子元素設置上外邊距(margin-top)時,也會發生外邊距重疊問題,意即子元素設置的上外邊距也會設置給父元素,如父元素也設置上外邊距時,一樣會有外邊距重疊問題,如要子元素僅在父盒子中移動位置,可依下列解決方案設置:
    1.為父元素設置1px的邊框
    2.為父元素設置padding:1px;
    3.為父元素設置overflow:hidden;(建議使用)
- 盒子模型布局穏定性來區分,建議先使用寛度(width寛度剩餘法),其次使用內邊距(padding),最後再使用外邊距(margin),原因:
    1.margin會有外邊距合併,IE6以下margin加倍的bug,所以最後使用
    2.padding會影響盒子大小,需要進行加減計算
    3.width可以使用寛度剩餘法來處理

    float: left (左浮動) right(右浮動) none(不浮動默認值)
  -特性:
    1.脫離標準流
    2.不占位置
    3.會影響標準流  
  -目的:為了讓多個塊級元素在同一行顯示。
  -float(浮漏特)浮-漂浮(脫離標準流)、漏(不佔位置)、特(與父盒子搭配使用並轉換具有行內塊特性)

  -塊級及行內元素添加浮動後具有行內塊的特性(不會獨佔一行、可設置寛高、盒子的大小由內容決定)-->前提未設置寛度

- 清除浮動:清除浮動造成的影響
    -父元素未設置高度,子元素設置浮動後,會造成高度塌陷問題
    -清除浮動方法
        1.額外標籤法:在浮動盒子後面添加一個空盒子-->W3C推薦,但少用
            .clear{
                clear: both; 清除浮動造成的影響
            }
        2.父元素添加overflow屬性-->觸發BFC
            overflow: hidden;
        3.使用after偽元素清除浮動-->為父元素添加下列設置
            .clearfix:after{
                content:".";  內容中有一個小點,不加,則舊版本瀏覽器會有空隙
                display: block;
                height: 0;
                visibility: hidden; 隠藏盒子
                clear: both;
            }
            .clearfix{  IE6、7清除浮動方法
                *zoom: 1; *代表IE6、7能識別的特殊符號,帶有*屬性,只有IE6、7才執行 zoom是IE6、7清除浮動方法
            }
        4.使用before和after雙偽元素清除浮動-->建議使用,固定加上clearfix
            .clearfix:before, .clear:after{
                content: "";
                display: table;
            }
            .clearfix:after{
                clear: both;
            }
            .clearfix{  IE6、7清除浮動方法
                *zoom: 1;
            }
- 定位:主要功能用來移動元素位置,屬性包含定位模式及邊偏移兩部份
    1.邊偏移屬性值:
      -top:頂端偏移量,定義元素相對於父元素上邊線的距離
      -bottom:底部偏移量,定義元素相對於父元素下邊線的距離 
      -left:左偏移量,定義元素相對於父元素左邊線的距離
      -roght:右偏移量,定義元素相對於父元素右邊線的距離
    2.定位模式:position:
      -static:默認值,自動定位,靜態定位,就是預設的標準流模式,邊偏移設置無效,一般用來清除定位模式
      -relative:相對定位(佔有位置,不脫標)
        特點:
        (1)相對於本身在原文檔流位置的左上角為基準點進行移動位置(相對於自己原來位置來移動位置)
        (2)原來位置繼續仍佔有,不脫離文檔流(相對定位不脫標)
      -absolute:絕對定位,相對於最近一個已經定位的父元素進行偏移(不佔有位置,完全脫標)
        特點:
        (1)脫離標準流,不佔位
        (2)父元素如未定位,則以瀏覽器窗口左上角為基準進行移動偏移
           父元素有定位時,則設置絕對定位的元素會以離它最近已經定位(絕對、相對或固定定位)的父元素(祖先)進行定位
           子絕父相:一般為子元素設置絕對定位,父級要設置相對定位-->利用相對定位特點佔有位置,不脫標及絕對定位不佔
                  有位置,完全脫標等特性
        (3)利用絕對定位的水平、垂直居中:
         1.設置絕對定位後,margin:0 auto就無效了
         2.水平居中: left: 50%;
                    margin-left: -自身盒子寛度的一半;
         3.垂直居中: top: 50%;
                    margin-top: -自身盒子高度的一半;
      -fixed:固定定位,相對瀏覽器窗口進行定位(是絕對定位的一種特殊情形)
         1.總是以瀏覽器窗口為基準.與父元素没有任何關係,只認瀏覽器
         2.固定定位完全脫離標準流,不佔有位置,且不隨滾動條滾動
         3.常用來固定導航欄、廣告、往上或回上一頁圖標
         4.固定定位的盒子一定要寛和高,除非有內容撐開不用寫
    
    3.z-index層次疊加
      在css中想要調整重叠定位元素的堆疊順序,可對定位元素應用z-index屬性改變層疊層級,可為正數、負數或0
      1.z-index默認值為0,值越大,越居上,如困相同時,依書寫順序,後來居上。
      2.數字後面不可加單位。
      3.只有相對定位、絕對定位、固定定位有此屬性,其餘標準流、浮動或靜態定都無此屬性。
    4.定位模式轉換
      1.與浮動相同,元素添加絕對定位與固定定位後,元素模式也會發生轉換為行內塊模式(寛高由內容撐開)
      2.行內元素(ex:a 或span)如設置絕對定位或固定定位後,可以不用轉換模式,直接給寛高即可(因為已轉換為行內塊元素,可設置寛高。
元素的顯示與隠藏
    display:none-->不佔有位置           block-->顯示
    visibility:hidden -->佔有位置 隠藏  visible-->對象顯示
    overflow:hidden  -->溢出隠藏        auto -->超出自動顯示滾動條,不超出不顯示滾動條   visible-->全部顯示
